# vue 组件缓存 router-view 详解

# 使用方式

利用meta属性

路由参数:

export default[
 {
  path:'/',
  name:'home',
  components:Home,
  meta:{
    keepAlive:false //需要被缓存的组件
 },
 {
  path:'/A',
  name:'A',
  components:A,
  meta:{
     keepAlive:true //不需要被缓存的组件
 } 
 {
  path:'/B',
  name:'B',
  components:A,
  meta:{
     keepAlive:false //不需要被缓存的组件
 } 
]

App.vue的template标签内：

<keep-alive>
  <router-view v-if="this.$route.meat.keepAlive"></router-view>
  <!--这里是会被缓存的组件-->
</keep-alive>
<keep-alive v-if="!this.$router.meta.keepAlive"></keep-alive>
<!--这里是不会被缓存的组件-->

# 问题

如以上首页 -> A -> B : 我们要实现首页到A页面会刷新页面，A到B，B返回A时，A需要保持之前的状态，即走缓存。

但是实际情况是：第一次首页到A页面会刷新页面，之后的每次A页面都会走缓存，比如：

​ 第一次首页搜索“苍井空”进入A页面，这时候A页面是有关苍老师的内容的没问题；

​ 从A页面进入B页面，B页面再返回A页面，A页面缓存了苍老师内容也没问题；

​ 但是从A页面回退到首页，首页再输入“石原里美”进入A页面，A页面还是显示苍老师的页面，这时候就出问题了，A页面一直停留在苍老师的内容里出不来了。。。。

# 解决

# 1.利用berforeRouteEnter实现前进刷新，后退缓存

在A页面添加beforeRouteEnter

beforeRouteEnter(to, from, next){
    next(vm => {
        //如果是"/"首页过来的执行以下操作，如果是B页面过过来就走缓存
        if(from.path == "/"){  
            //1.会导致：页面先显示原来缓存页面，然后再刷新
            window.location.reload();
            //2.通过.$forceUpdate()强制刷新，未知原因不起效果
            vm.$forceUpdate();
            //3.重新执行页面初始化需要执行的逻辑，会出现但首次加载A页面时即执了两次mounted();原本页面执行一次，beforeRouteEnter里又执行一次，页面初始化的内容请求一般来说会比较多，如果这时候多做一次无用请求那么会给服务器造成一定的压力。
            //解决方式:直接将mounted内容写在此处，删除mounted的代码。这又会导致在本页面下刷新A本页面时，不会执行beforeRouteEnter，A页面就没有初始化逻辑了。
            vm.mounted();
        }
    })
},

注意：实际上这个页面除了第一次进入和刷新页面外都是走缓存的，即除了第一进入和刷新页面会执行mounted，其他其余走缓存的情况进入都不会执行mounted，但是activated不一样，无论有没有走缓存，都会执行。由于beforeRouteEnter本页面刷新不执行，所以可以把activated替换beforeRouteEnter使用。有兴趣看一下mounted与activated的执行 (opens new window)

# 2.利用beforeRouteLeave加activated的方式实现前进刷新，后退缓存

给A页面的meta新加一个标志位refreshing，用来判断A页面状态：true就刷新，false走缓存

 {
  path:'/A',
  name:'A',
  components:A,
  meta:{
     keepAlive:true ,//不需要被缓存的组件
     refreshing: true,
 }

首页加beforeRouteLeave，如果首页前往的是A页面，就把A页面的refreshing设为true

 beforeRouteLeave(to, from, next){
            if(to.path == '/A'){
                to.meta.refreshing = true;
            }
            next(true);
        },

B页面加beforeRouteLeave，如果B页面前往的是A页面，就把A页面的refreshing设为false

beforeRouteLeave(to, from, next){
            if(to.path == '/A'){
                to.meta.refreshing = false;
            }
            next(true);
        },

A页面的activated代码，按照refreshing状态决定是否执行 页面初始化逻辑的代码

activated(){
            if(this.$route.meta.refreshing){
               ...//页面初始化逻辑
            }
        },
