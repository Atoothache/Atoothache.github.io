# flyway使用

1.下载：https://repo1.maven.org/maven2/org/flywaydb/flyway-commandline/4.2.0/

2.解压

创建表空间和用户

--初始化用户
create user SJBAT
  identified by 123456
  default tablespace SYSTEM
  temporary tablespace TEMP
  profile DEFAULT;
-- Grant/Revoke role privileges 
grant connect to SJBAT;
grant resource to SJBAT;
-- Grant/Revoke system privileges 
grant select any sequence to SJBAT;
grant select any table to SJBAT;
grant create database link to SJBAT;
grant create any synonym to SJBAT;
grant create any sequence to SJBAT;
grant create any view to SJBAT;
grant create any procedure to SJBAT;
grant unlimited tablespace to SJBAT;

flyway 配置文件：

flyway.url=jdbc:oracle:thin:@192.168.133.13:1521:orcl

flyway.driver=oracle.jdbc.driver.OracleDriver

flyway.user=WK_NAN

flyway.password=dragon123

flyway.baselineOnMigrate: true
flyway.placeholderReplacement: false

flyway最基本的几个命令。

Migrate：应用所有的迁移到最新版本，它会在你的DB中新建个表schema_version来存放每次升级的版本信息。

Clean：clean all objects

Info：打印所有的迁移的信息以及状态。

Validate：迁移之前进行验证。

Baseline：初始化schema_version表，并插入一条原始verion=1。

Repair：它主要做了两件事，移除所有失败的迁移（升级），重置校验和。

1，migrate
执行sql文件夹中的sql，更新到最新版本
2，baseline
对已经拥有表结构的数据库中实现添加Metadata表。注意：sql脚本执行，会从第二个V脚本开始，因为这个命令会以当前数据库版本为基线进行版本创建。
3，clean
清除掉对应数据库Schema中所有的对象，说白了，删库。但在生产环境务必禁用。
4，info
打印所有的Migrations的详细和状态信息。
5，repair
repair操作能够修复metaData表。
6，undo 撤销操作，社区版不支持。
7，validate 验证已经apply的Migrations是否有变更，默认开启的，原理是对比MetaData表与本地Migrations的checkNum值，如果值相同则验证通过，否则失败。
